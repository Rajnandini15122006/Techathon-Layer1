# Layer 3: HRVC Risk Engine 🟡

## Overview
The HRVC (Hazard-Risk-Vulnerability-Capacity) Risk Engine calculates flood risk scores for each grid cell using a simplified, scientifically-grounded formula.

## Risk Formula

```
Risk Score = (Hazard × Exposure × Vulnerability) ÷ Capacity
```

All components are normalized to 0-100 scale, with the final risk score capped at 100.

## Components

### 1. Hazard (H)
Measures the physical threat level:
- **Rainfall Forecast** (40%): Predicted rainfall in mm
- **River Level** (35%): Water level in meters
- **Soil Saturation** (25%): Percentage of soil moisture

### 2. Exposure (E)
Measures what is at risk:
- **Population Density** (60%): People per km²
- **Traffic Density** (40%): Vehicles per hour

### 3. Vulnerability (V)
Measures susceptibility to damage:
- **Slum Percentage** (40%): Informal settlements %
- **Elderly Percentage** (30%): Population over 65 %
- **Low Elevation Percentage** (30%): Area below flood threshold %

### 4. Capacity (C)
Measures ability to cope:
- **Shelter Count** (40%): Number of emergency shelters
- **Hospital Beds** (35%): Available medical capacity
- **Drain Strength** (25%): Drainage infrastructure quality (0-100)

## Risk Levels

| Score Range | Level | Color | Action Required |
|-------------|-------|-------|-----------------|
| 80-100 | CRITICAL | Red | Immediate evacuation |
| 60-79 | HIGH | Orange | Prepare for evacuation |
| 40-59 | MEDIUM | Yellow | Monitor closely |
| 20-39 | LOW | Green | Normal monitoring |
| 0-19 | MINIMAL | Blue | No action needed |

## API Endpoints

### Calculate Grid Risks
```
GET /api/risk/calculate
```

Parameters:
- `lat_min`, `lat_max`: Latitude bounds
- `lon_min`, `lon_max`: Longitude bounds
- `use_synthetic`: Use synthetic data (default: true)

Response:
```json
{
  "status": "success",
  "total_cells": 100,
  "grid_cells": [
    {
      "cell_id": 1,
      "latitude": 18.45,
      "longitude": 73.80,
      "risk_score": 75.23,
      "risk_level": "HIGH",
      "components": {
        "hazard": 82.5,
        "exposure": 68.3,
        "vulnerability": 45.2,
        "capacity": 55.8
      }
    }
  ],
  "summary": {
    "avg_risk": 45.67,
    "max_risk": 89.34,
    "critical_cells": 5,
    "high_risk_cells": 12
  }
}
```

### Get Ward Priorities
```
GET /api/risk/ward-priorities
```

Parameters: Same as above

Response:
```json
{
  "status": "success",
  "total_wards": 10,
  "ward_priorities": [
    {
      "ward_id": 5,
      "ward_name": "Ward 5",
      "priority_rank": 1,
      "avg_risk": 78.45,
      "max_risk": 92.3,
      "cell_count": 10,
      "high_risk_cells": 7,
      "priority_level": "HIGH"
    }
  ]
}
```

### Calculate Single Cell Risk
```
POST /api/risk/calculate-single
```

Body: Cell data with all HRVC fields

## Ward Priority List

The ward priority list is generated by:
1. Calculating risk scores for all grid cells
2. Aggregating cells by ward_id
3. Computing average risk per ward
4. Sorting wards by average risk (descending)
5. Assigning priority ranks

This replaces the need for a separate "Ward Priority" module - it's now a dashboard feature derived from grid-level risk calculations.

## Usage Example

```python
from app.services.risk_engine import RiskEngine

engine = RiskEngine()

# Calculate risk for a single cell
cell_data = {
    'rainfall_mm': 150,
    'river_level_m': 5.2,
    'soil_saturation_pct': 85,
    'population_density': 25000,
    'traffic_density': 500,
    'slum_percentage': 12,
    'elderly_percentage': 15,
    'low_elevation_percentage': 40,
    'shelter_count': 3,
    'hospital_beds': 150,
    'drain_strength': 65
}

risk_data = engine.calculate_risk_score(cell_data)
print(f"Risk Score: {risk_data['risk_score']}")
print(f"Risk Level: {risk_data['risk_level']}")
```
