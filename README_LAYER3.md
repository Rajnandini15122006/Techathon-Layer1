# Layer 3: HRVC Risk Engine - Quick Start

## What is Layer 3?

Layer 3 implements the HRVC (Hazard-Risk-Vulnerability-Capacity) Risk Engine that calculates flood risk scores for each grid cell using the formula:

```
Risk = (Hazard × Exposure × Vulnerability) ÷ Capacity
```

## Quick Test

Run the test script to verify the risk engine:

```bash
python test_risk_engine.py
```

This will:
- Test high-risk and low-risk scenarios
- Generate a synthetic grid with risk data
- Calculate ward priorities
- Show overall statistics

## Start the Server

```bash
python run_local.py
```

## Access the Dashboard

Open your browser to:
```
http://localhost:8000/static/risk_dashboard.html
```

Or click the "🟡 Risk Dashboard" button in the main interface.

## API Endpoints

### Calculate Risk Scores
```bash
curl "http://localhost:8000/api/risk/calculate?lat_min=18.45&lat_max=18.55&lon_min=73.80&lon_max=73.90"
```

### Get Ward Priorities
```bash
curl "http://localhost:8000/api/risk/ward-priorities?lat_min=18.45&lat_max=18.55&lon_min=73.80&lon_max=73.90"
```

### Calculate Single Cell Risk
```bash
curl -X POST http://localhost:8000/api/risk/calculate-single \
  -H "Content-Type: application/json" \
  -d '{
    "rainfall_mm": 150,
    "river_level_m": 5.2,
    "soil_saturation_pct": 85,
    "population_density": 25000,
    "traffic_density": 500,
    "slum_percentage": 12,
    "elderly_percentage": 15,
    "low_elevation_percentage": 40,
    "shelter_count": 3,
    "hospital_beds": 150,
    "drain_strength": 65
  }'
```

## Risk Components

### Hazard (H)
- Rainfall forecast (mm)
- River level (m)
- Soil saturation (%)

### Exposure (E)
- Population density (per km²)
- Traffic density (vehicles/hour)

### Vulnerability (V)
- Slum percentage (%)
- Elderly percentage (%)
- Low elevation percentage (%)

### Capacity (C)
- Shelter count
- Hospital beds
- Drain strength (0-100)

## Ward Priority List

The ward priority list is automatically generated by:
1. Calculating risk for all grid cells
2. Aggregating cells by ward
3. Computing average risk per ward
4. Sorting by risk (highest first)
5. Assigning priority ranks

No separate module needed - it's a dashboard feature!
